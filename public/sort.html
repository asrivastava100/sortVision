Running time complexity analysis

  • The Bubble Sort algorithm has a worst case running time of O(n2), where n is the size of the array.
  • It has a best case running time of O(n), where n is the size of the array. This occurs when the array is already sorted.
  • The space complexity of Bubble Sort is constant - O(1)

    • Running time complexity analysis

  • The Selection Sort algorithm has a worst case running time of O(n2), where n is the size of the array.
  • The space complexity of Selection Sort is constant - O(1)

    • Running time complexity analysis

  • The Merge Sort algorithm has a worst case running time of O(nlogn), where n is the size of the array.
  • It has a best case running time is also O(nlogn), where n is the size of the array.
  • The space complexity of conventional Merge Sort is linear - O(n)
  • We should note that there is an In-place Merge Sort algorithm which has constant space complexity - O(1). Too good to be true? One of the implementations of In-place Merge Sort has a worst case running time of O(nlog2n)
  • There are many implementations for In-place merge sort and care must be taken when choosing an implementation as there are other factors to consider. The reader is welcome to look at the other resources tab above.
  • Overall which implementation one chooses depends on the particular preferences i.e. optimising space over performance (In-place) or the other way around (Merge Sort with temporary array).
